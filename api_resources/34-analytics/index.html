API Analytics

Analytics

/environments/{environmentName}/stats
Reports on metrics for API usage and performance
Returns metrics based on specified dimensions. Dimension: The dimension selected from the list of dimension recorded by Apigee. In this example the dimension is API name. Metrics: The metric for the chosen dimension. In this example the metric message_count. Time range: In this example the time range is July 23 to July 24, 2012. Timestamp is in Unix time (seconds since January 1, 1970, Midnight GMT.) The dimension is the entity for which Apigee collects metrics. The 'dimension' that is set for this report is 'apis'. There is long list of dimensions that can be used in your reports. Some dimensions of inital interest are: client_ip: The IP address of the app making the request to your API developer: The developer of the app making the request to your API developer_app: The name of the app making the request to your API access_token: The app end user's OAuth access token The function run on the metric in this example is 'sum'. The other functions you can use are sum, average, min, and max. The metric in this request is 'message_count'. Other metrics you can use are: target_response_time: How long it takes the backend service to respond response_status_code: The status code of the response, which is good for measuring specific errors message_count: Total number of messages request_size: Size if the request message response_processing_latency: Duration of processing by the Apigee response pipeline response_size: Size of the response message request_processing_latency: Duratio nof processing by the Apigee request pipeline. total_response_time: Overall duration of processing for Apigee plus the backend service. is_error: Defines a message as an error
Get API StatisticsDetail 
/environments/{envname}/stats/apis?select={metrics}&timeRange={startDate~endDate}&timeUnit={unit}&sortby={metrics}&sort={order}
Verb:GET
Description:Use the GET method to retrieve metrics for all the APIs for an organization and environment.
Auth:Basic Auth
Get Statistics for an EnvironmentDetail 
/environments/{environmentName}/stats/?select={metrics}&timeRange={startDate~endDate}&timeUnit={unit}
Verb:GET
Description:Gets statistics for a specified time range for the environment specified. Available staistics are message count, error count, and API count.
Auth:Basic Auth
Get Statistics for Top APIsDetail 
/environments/{environmentName}/stats/apis
Verb:GET
Description:Returns 5 APIs with the highest number of active end users (app end users with throughput) in the last 24 hours.
Auth:Basic Auth
Get Statistics for top AppsDetail 
/environments/{environmentName}/stats/apps
Verb:GET
Description:Use the GET method to retrieve the metrics for the first five apps for a given customer in a specific organization and environment.
Auth:Basic Auth
Get Statistics for top DevelopersDetail
