Name the API product
Bundle Resources by defining resource paths (URIs)
Define the service plan settings for quota (permitted requests per day/month) and/or spike arrest (permitted requests per second/minute).  The service plan defines how much your developers can consume over time. At runtime, these variables are read by the Rate Limiting Policy applied to an API or Resource. Rate Limiting Variabes are set as API Product attributes:
developer.quota.limit - integer
developer.quota.interval - integer
developer.quota.timeunit second/minute/hour/day/month)
You can also use attributes to set any arbitrary metadata that you need for monitoring or analytics.
